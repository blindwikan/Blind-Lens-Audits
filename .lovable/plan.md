
## Goal

Let users download their Blind Lens audit as an accessible, branded PDF directly from the results page. Entirely client-side, no backend changes, no changes to audit logic.

## Library

- Install `@react-pdf/renderer` (npm).
- Use its `<Document>`, `<Page>`, `<View>`, `<Text>`, `<Link>` primitives — never images of text.
- Use `PDFDownloadLink` for one-click download (button only renders after audit data exists).

## New files

- `src/lib/pdf/BlindLensReportPdf.tsx` — the PDF document component.
- `src/lib/pdf/pdfTheme.ts` — colors, fonts, spacing tokens mirroring the live dark theme (Navy bg `#0F172A`-ish, teal accent, high-contrast body text). Map verdict tone → hex (success/accent/warning/destructive) so the score color matches the site.
- `src/lib/pdf/scoreUtils.ts` — extract `calculateBlindLensScore` + `getScoreVerdict` from `ResultsPage.tsx` so both the page and the PDF share one source of truth (no logic change, just relocation + re-import in `ResultsPage.tsx`).

## Modified files

- `src/components/ResultsPage.tsx`
  - Import the score helpers from the new shared module.
  - Add a new `<motion.section>` directly above the "Want to fix this?" CTA containing a `PDFDownloadLink` styled as the existing primary button (`bg-primary text-primary-foreground … rounded-xl glow-primary`, `Download` lucide icon).
  - Filename built from `new URL(result.url).hostname.replace(/\./g, '-')` and today's date in `YYYY-MM-DD`. Pattern: `blind-lens-audit-{domain}-{date}.pdf`.
  - Button only rendered when `result` is present (already true on this page).
  - No other layout changes.

## PDF structure

Document props: `title="Blind Lens Accessibility Audit — {domain} — {date}"`, `language="en"`, `author="Blind Lens"`.

Pages (react-pdf auto-paginates; explicit `break` where needed):

1. **Cover page**
   - Blind Lens wordmark/logo at top (text-based wordmark to keep it accessible — no image of text).
   - Title "Accessibility Audit Report".
   - Audited URL.
   - Audit date + time (locale string).
   - Large color-coded Blind Lens Score number with verdict label underneath.
   - Immediately adjacent SR-equivalent `<Text>`: "Blind Lens Score: {n} out of 100. Verdict: {label}." (visually small/muted but present in reading order).

2. **Wikan's Verdict** — heading + the `closingSummary` quoted, prominent serif-ish styling consistent with the site's italic blockquote.

3. **WAVE Findings**
   - Summary row: Errors, Contrast Failures, Alerts, Positive Features, Structural Elements (each a labeled number block).
   - Then each issue: severity tier label (Red Flag / It's Complicated / Minor Ick) + title + Wikan's first-person `blindLensCommentary` verbatim. No rewording.
   - Attribution line: "WAVE accessibility data provided by WebAIM (wave.webaim.org)".

4. **Additional Accessibility Checks (Lighthouse)** — only rendered if `lighthouse?.failedAudits.length`. Compact list of `title` + optional `displayValue`, matching the live accordion content (flat list in PDF).

5. **Footer (fixed on every page)** — `<View fixed>` at bottom: muted text "Generated by Blind Lens — See your website through the blind lens. blind-lens-audits.lovable.app" on the left, page number via `render={({ pageNumber, totalPages }) => …}` on the right.

Reading order matches visual order top-to-bottom on every page (single column layout, score block followed by SR text, then sections in listed order).

## Visual design

- Page background: dark navy matching `--background`.
- Body text: high-contrast off-white.
- Section headings: bold, larger, teal accent underline or left border.
- Score: large numeral colored by verdict (`text-success` / `text-accent` / `text-warning` / `text-destructive` mapped to hex).
- Generous padding (`Page` padding ~48pt; section gap ~24pt).
- Use built-in `Helvetica` family (no font registration required) to keep bundle small and avoid font-loading races; sizes/weights chosen to feel like Space Grotesk/Inter hierarchy.

## Accessibility checklist enforced in code

- `<Document title=… language="en">`.
- All content in `<Text>` — no `<Image>` for textual content.
- Score followed immediately by SR-readable equivalent text in the same `<View>`.
- Logical top-to-bottom flow per page; no absolute positioning that reorders reading.
- Links via `<Link src=…>` with descriptive text.

## Verification steps (after implementation)

1. Run a real audit on `https://twoblindbrothers.com` in the preview.
2. Click "Download Report"; confirm filename `blind-lens-audit-twoblindbrothers-com-{today}.pdf`.
3. Open PDF: cover → Wikan's Verdict → WAVE → Lighthouse → footer on every page.
4. Confirm score color matches verdict, Wikan's first-person commentary is intact.
5. Inspect PDF metadata (title + language) via `pdfinfo` or a reader's properties dialog.

## Out of scope

- No edits to `accessibility-audit` edge function, `audit.ts` API layer, or scoring math.
- No layout changes to `ResultsPage.tsx` beyond inserting the download button section.
- No auth, gating, analytics, or backend storage of the report.
